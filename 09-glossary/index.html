    용어 사전🔗

    매뉴얼에 자주 나오는 단어를 셀러 입장에서 풀어 정리.

    ㄱ🔗

    가격 단위 (priceUnit) 판매가의 마지막 자리를 어디까지 정렬할지. 예) 100원 단위면 12,348 → 12,400. 쿠팡·11번가는 10원 단위 미만 거부.

    가격 정책 "얼마에 팔지" 한 번 정하면 모든 상품에 자동 적용되는 규칙. 마진율·최소 마진·수수료·할인 등 여러 변수 조합. 자세히는 7. 가격 정책.

    그룹 상품 같은 후드티에 색상만 다른 변형(검정·흰색·회색)을 하나로 묶어 등록. 스마트스토어 검색 결과에 대표 1개만 나오고 색상 미리보기가 같이 표시 → 노출 효과 ↑.

    ㄴ🔗

    노출 검색 결과·카테고리 페이지·기획전 등에서 고객이 상품을 보는 횟수. 노출 ↑ → 클릭 ↑ → 매출 ↑.

    노출 풀 검색 결과에 들어갈 자격이 있는 상품들 그룹. 카테고리·KC·옵션 형식이 잘못되면 풀에서 빠짐.

    ㄷ🔗

    다중 매핑 / Bulk 매핑 여러 상품의 카테고리·옵션을 한 번에 처리. 신규 셀러가 기존 마켓 상품을 가져올 때 유용.

    다크모드 화면을 어두운 배경 + 밝은 글씨로 보는 모드. 매뉴얼 우측 상단 ⚙️ 또는 ☀️/🌙 토글로 전환.

    ㅁ🔗

    마진율 (marginRate) 원가의 몇 %를 추가로 받을지. 예) 30% → 원가 10,000원이면 13,000원.

    매장 ID 롯데온 셀러센터에서 받는 셀러 식별 번호. 모든 상품에 자동 결합.

    멀티 계정 한 셀러가 여러 마켓 계정을 동시에 운영하는 것. Lonit은 마켓별로 여러 계정 등록 가능.

    모델명 같은 시리즈끼리 묶이는 식별자. 스마트스토어 SEO에 도움.

    ㅂ🔗

    발주 정책 롯데온에서 모든 상품에 결합해야 하는 출고·배송·AS·반품 규칙. 한 번 설정하면 자동 적용.

    브랜드 검증 스마트스토어에서 셀러가 등록한 브랜드가 네이버 DB에 있는 정식 브랜드인지 확인 + 검색 가중치.

    바이트 (byte) 글자 길이 단위. 한글 1자 ≈ 3바이트. 11번가 상품명 100바이트 = 한글 33자 정도.

    ㅅ🔗

    상품명 SEO 검색에 잘 잡히도록 상품명을 다듬는 것. 핵심 키워드 + 자연스러운 한국어 + 금지어 회피.

    서버 IP Lonit이 마켓 API 호출할 때 사용하는 고정 IP. 마켓 셀러센터에 미리 등록해야 거부되지 않음. 네이버·쿠팡·11번가는 158.247.214.201. 롯데온은 별도 환경 가능.

    송장 택배 추적 번호 + 택배사. Lonit에 한 번 입력하면 4마켓에 자동 분배.

    수집 무신사·SSG 같은 소싱처 페이지에서 상품 정보(이름·가격·옵션·이미지)를 가져오는 것. Lonit 익스텐션이 자동.

    스케줄러 정해진 주기로 자동 실행되는 작업 시스템. 가격·재고·주문 동기화는 스케줄러가 돌림.

    ㅇ🔗

    옵션 / 옵션값 색상·사이즈 같은 상품 변형. "검정/L"에서 검정·L이 옵션값.

    원가 / 매입가 셀러가 무신사 등에서 상품을 가져오는 가격. 정책 계산의 시작점.

    익스텐션 (확장프로그램) Chrome 브라우저에 설치하는 도구. Lonit은 12개 사이트에서 상품을 자동으로 가져오는 익스텐션 제공.

    임시저장 쿠팡 등에서 상품 등록 후 셀러 승인 요청 전 단계. Lonit이 일괄 승인 요청 지원.

    ㅈ🔗

    작업 공간 / 테넌트 셀러별 데이터 공간. 다른 셀러에게 영업 데이터(상품·주문 등)는 공개되지 않음.

    전체 동기화 가격·재고·옵션 모두 한 번에 마켓에 반영. 보통 자동, 강제 동기화도 가능.

    정책 적용 한 번 만든 정책을 상품에 부착. 카테고리별·검색 필터별·상품별로 다른 정책 적용 가능.

    ㅋ🔗

    카탈로그 매칭 (쿠팡) 같은 상품끼리 묶이는 쿠팡 페이지에 합류. 매칭되면 즉시 노출 풀 진입, 매칭 실패하면 새 카탈로그를 만들어야 해서 노출 약함.

    카테고리 매핑 무신사 "후드티" → 스마트스토어 "패션의류 → 상의 → 후드티" 처럼 다른 마켓 카테고리로 변환. Lonit은 다단계 자동 매핑.

    큐 (Queue) 업로드·동기화 작업이 대기하는 줄. Lonit은 마켓별 독립 큐로 처리 → 한 마켓 느려도 다른 마켓 영향 없음.

    클레임 주문 후 발생하는 취소·반품·교환 요청. Lonit이 4마켓 통합 화면에서 처리.

    ㅌ🔗

    태그 상품을 분류하는 키워드. 스마트스토어는 태그가 검색 매칭에 큰 영향. Lonit이 네이버 인기 태그 자동 추천.

    토들러 / 키즈 사이즈 2T, 3T 등 영유아용 사이즈. 쿠팡은 별도 옵션 슬롯.

    ㅍ🔗

    판매가 (sell_price / slPrc) 실제 고객에게 보이는 가격. 정책에 따라 자동 계산.

    페이로드 (Payload) 마켓 API에 보내는 데이터 묶음. Lonit이 마켓별 형식에 맞게 자동 변환.

    프리플라이트 (Preflight) 업로드 전 검증. 쿠팡은 SKU 사전 검색해서 기존 카탈로그 합류 여부 확인.

    ㅎ🔗

    할인율 (discountRate) / 할인 금액 (discountAmount) 판매가에서 추가로 깎는 비율 또는 금액. 마진은 그대로 두고 노출 효과 ↑.

    활동 센터 업로드·동기화·수집 등 작업 진행 상황을 한눈에 보는 화면.

    A-Z🔗

    API Application Programming Interface. 프로그램끼리 데이터를 주고받는 약속. Lonit은 마켓 API로 상품 등록·동기화.

    DB (Database) 데이터를 저장하는 시스템. 셀러 입장에서는 신경 쓸 일 거의 없음.

    HMAC 쿠팡 API 인증 방식. Lonit이 자동 처리.

    KC 인증 국내 안전 인증. 영유아·전자제품 등 일부 카테고리 필수. 11번가는 카테고리에 따라 인증번호 입력 필요.

    OAuth2 스마트스토어·롯데온 API 인증 방식. 키 발급 후 Lonit에 등록.

    SaaS Software as a Service. 설치 없이 웹에서 바로 쓰는 서비스. Lonit이 SaaS.

    SEO Search Engine Optimization. 검색 결과 상위 노출. 스마트스토어에서 가장 중요.

    SKU 상품을 식별하는 코드. 쿠팡 카탈로그 매칭의 핵심.

    T사 Lonit과 비슷한 일을 하는 다른 SaaS 도구. 3. T사와 비교에서 자세히.
