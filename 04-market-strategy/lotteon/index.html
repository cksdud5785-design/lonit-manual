    롯데온 노출 전략🔗

    정책이 곧 노출. 매장 정책 + 발주 정책 + 카테고리가 다 맞아야 합니다.

    롯데온

    1. 롯데온이 다른 마켓과 다른 점🔗

    flowchart LR subgraph OTHER_MKT["스마트스토어·쿠팡·11번가"] N1[상품 정보] N1 --> N2[등록] N2 --> N3[노출] end subgraph LOTTE["롯데온"] L1[상품 정보] L1 --> L2{매장 정책 결합} L2 --> L3{발주 정책 결합} L3 --> L4[등록] L4 --> L5[노출 + 정책 점수] end style L2 fill:#fce7f3,stroke:#ed1c24 style L3 fill:#fce7f3,stroke:#ed1c24

    롯데온은 상품 + 정책의 조합으로 등록합니다. 정책 설정이 잘못되면 발주가 거부됩니다.

    2. 매장 ID — 가장 먼저 확인🔗

    각 셀러는 매장 ID를 받습니다. 이 ID가 없으면 등록 자체가 안 됩니다.

    flowchart LR USER[롯데온 셀러] --> 등록[Lonit에
    매장 ID 입력] 등록 --> 자동[Lonit이 모든
    상품에 자동 적용] style 등록 fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style 자동 fill:#dcfce7,stroke:#22c55e

    매장 ID는 롯데온 셀러센터 → 매장 관리 에서 확인 가능합니다.

    3. 가격 — 판매가 (slPrc) 기준 등록🔗

    롯데온은 등록·갱신 시 판매가(slPrc) 를 기준으로 가격을 보냅니다. 다른 마켓처럼 "정가 + 할인" 분리 자동화는 현재 Lonit에서 별도로 지원하지 않습니다 (롯데온 정책 변경에 따라 향후 확장 예정).

    flowchart LR Calc[정책 적용 후
    판매가 계산] --> Send[slPrc 단일 값으로
    롯데온 등록·갱신] Send --> Display[고객 화면에
    판매가 표시] style Send fill:#fce7f3,stroke:#ec4899

    할인 표시를 원한다면

    롯데온 셀러센터에서 직접 프로모션 / 즉시할인 / 카드사 행사를 별도 등록할 수 있습니다. Lonit이 등록한 판매가 위에 추가 할인이 적용됩니다.

    3-1. Lonit 정책의 할인 변수 활용🔗

    가격 정책의 discountRate·discountAmount는 판매가 자체를 낮추는 방식으로 동작합니다 — 즉, 정가/할인가 분리가 아니라 최종 slPrc가 작아집니다. 정가 표시 + 할인율 배지를 원한다면 셀러센터에서 별도 설정.

    4. 카테고리 — 4단계🔗

    롯데온은 4단계 카테고리 입니다.

    패션 / 남성 / 상의 / 후드티

    스마트스토어·쿠팡보다 한 단계 얕음. Lonit이 자동 매핑합니다.

    5. 발주 정책 — 등록 거부의 가장 큰 원인🔗

    롯데온은 발주 정책을 모든 상품에 결합해야 합니다.

    5-1. 발주 정책에 들어가는 것🔗

    mindmap root((발주 정책)) 배송 배송비 0원·무료 반품·교환 배송비 (자동 6,000원) AS AS 전화번호 AS 안내문 재고 최대 재고 수량 (기본 10) 안전 재고 출고 출고 마감 시각 배송 소요일

    5-2. Lonit이 자동 적용🔗

    매장 ID를 등록한 상태에서 정책을 1번 만들면, 모든 상품에 자동 적용됩니다.

    항목 기본값 (Lonit 설정)
    배송비 무료 (3만원 이상 무료 등)
    반품 배송비 6,000원 (정책에서 변경 가능)
    교환 배송비 6,000원
    최대 재고 10개 (정책에서 변경 가능)
    AS 안내 "구매처 문의"
    출고 평일 14시 마감

    7. 가격 정책 챕터에서 정책 변경 방법 설명.

    5-3. 발주 거부 사유🔗

    사유 해결
    매장 ID 누락 설정 → 마켓 계정에서 매장 ID 입력
    정책 ID 누락 정책 매핑 자동 (Lonit이 처리)
    가격 0원 정책 마진 검토 (판매가 > 0 보장)
    배송지 미설정 출고지·반품지 자동 등록 (lotteon 셀러센터에서 1회 설정)

    6. 카드 혜택 — 노출에 영향🔗

    롯데온은 롯데카드·신한카드 등의 카드 혜택이 노출 가중치에 영향을 줍니다. Lonit은 이 부분을 자동으로 처리:

    • 카드 혜택 가격은 이미 적용된 가격으로 보임
    • Lonit이 즉시할인 + 카드 혜택을 분리해 등록
    • 셀러는 정책에서 할인율만 설정하면 됨
    flowchart LR A[원가 18,000] --> B{Lonit 자동} B --> C[정가: 30,000] B --> D[즉시할인: -5,000] B --> E[카드혜택: -2,000] B --> F[최종: 23,000] F --> G[롯데온 표시:
    ~30,000~ → 23,000
    23% 할인] style B fill:#fce7f3,stroke:#ec4899 style G fill:#dcfce7,stroke:#22c55e

    7. 옵션 — 2축까지 자동 변환🔗

    롯데온도 색상 + 사이즈 2축까지 정상 처리합니다.

    7-1. parentId 계층 자동 감지🔗

    flowchart TB Source[원본 상품
    색상별로 다른 SKU] --> Detect{Lonit 자동 감지} Detect -->|parentId
    계층 발견| Hier[계층 옵션
    색상 + 사이즈 2축] Detect -->|단일 SKU| Flat[1축 옵션] Hier --> Upload[롯데온 등록] Flat --> Upload style Hier fill:#dcfce7,stroke:#22c55e

    같은 상품의 색상 변형들을 자동으로 인식해 2축 옵션으로 등록합니다.

    8. 동기화 — 정책 변경 시 자동 반영🔗

    가격 정책을 바꾸면 롯데온 등록된 상품 가격도 5분 안에 자동 변경됩니다.

    sequenceDiagram USER->>Lonit: 정책 마진 30% → 35% Lonit->>Lonit: products.updatedAt bump Lonit->>Lonit: 자동 동기화 엔진
    변경 감지 Lonit->>LOTTE: 4,496건 가격 일괄 업데이트 LOTTE-->>Lonit: ✅ 정가 + 할인가 분리 적용

    9. 자주 발생하는 문제🔗

    9-1. "발주 거부"🔗

    대부분 매장 ID 또는 정책 ID 누락. 설정 → 마켓 계정 → 롯데온 에서 매장 ID 확인.

    9-2. "할인 표시 안 됨"🔗

    과거 슬PRC 정가/판매가 혼동 버그가 있었으나 현재 fix 완료. 만약 여전히 안 보이면 트러블슈팅 참고.

    9-3. "재고 100인데 마켓에는 10"🔗

    기본 정책의 최대 재고가 10. 정책에서 maxStock 을 100으로 늘리면 됨.

    📌 한 줄 요약

    롯데온 = 정책 + 매장 ID. 매장 ID 등록 + 발주 정책 1개 설정이 첫 관문. 가격은 slPrc 단일 기준 등록 (할인 표시는 셀러센터에서 별도 프로모션). 카테고리 4단계.

    10. 요약 체크리스트🔗

    롯데온 등록 잘 되려면:

    • 매장 ID 정확
    • 발주 정책 1개 설정 (배송비·AS·재고)
    • 출고지·반품지 셀러센터에서 1회 설정
    • 가격 정책 (정가 + 할인 자동 분리)
    • 카테고리 4단계 정확
    • 옵션 2축 (색상 + 사이즈)
