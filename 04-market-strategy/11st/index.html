    11번가 노출 전략🔗

    등록 형식 완성도가 곧 노출. KC 인증, 신상품 코드, 카테고리 — 하나라도 빠지면 안 됨.

    11번가

    1. 11번가의 노출 결정 요인🔗

    flowchart TB
    상품[등록 상품] --> Check1{등록 형식
    완성?}
    Check1 -->|아니오| Reject[❌ 등록 거부]
    Check1 -->|예| Check2{KC 인증
    제대로?}
    Check2 -->|아니오| Reject
    Check2 -->|예| Check3{카테고리
    4단계 정확?}
    Check3 -->|아니오| Pool_low[하위 풀 진입
    노출 ↓↓]
    Check3 -->|예| Check4{신상품 코드
    prdStatCd=01?}
    Check4 -->|아니오| Pool_normal[일반 풀
    노출 보통]
    Check4 -->|예| Pool_high[신상품 가중치
    노출 ↑]

    style Reject fill:#fee2e2
    style Pool_high fill:#dcfce7,stroke:#22c55e

    11번가는 형식 검증이 가장 까다롭습니다. 형식이 맞으면 노출, 안 맞으면 거부.

    2. KC 인증 — 등록 거부의 가장 큰 원인🔗

    KC 인증(국내 안전 인증)은 카테고리에 따라 필수입니다.

    2-1. KC 인증 필요한 카테고리 (예시)🔗

    카테고리 KC 인증 필요?
    의류 일부 (영유아·아동 등)
    신발 영유아 신발만
    전자제품 거의 모두 ✅
    화장품 일부
    식품 별도 인증
    가방 보통 X

    2-2. Lonit이 자동 처리🔗

    flowchart LR
    카테고리[카테고리 매핑됨] --> Need{KC 필요?}
    Need -->|예| Auto[Lonit 자동 입력
    정책에서 받은 값]
    Need -->|아니오| Skip[자동 생략]

    Auto --> Upload[11번가 등록]
    Skip --> Upload

    style Auto fill:#fce7f3,stroke:#ec4899

    2-3. 셀러가 해야 할 일🔗

    KC 인증 정보(인증번호, 인증기관)는 정책 → 11번가 인증 정보 에서 1번 입력해두면 됩니다.

    항목 입력
    KC 인증번호 셀러가 발급받은 번호 (또는 면제 사유)
    인증기관 한국기계전기전자시험연구원 등
    인증연도 YYYY

    3. 신상품 코드 — 신상 가중치🔗

    11번가는 신상품 코드(prdStatCd=01)를 받은 상품에 노출 가중치를 더 줍니다.

    3-1. 헷갈리는 부분🔗

    flowchart LR
    Old[과거 오해] -->|prdStatCd=판매상태| Wrong[X 잘못된 이해]
    Now[현재 명확] -->|prdStatCd=01
    = 신상품 condition| Right[✅ 신상품 가중치]

    style Right fill:#dcfce7,stroke:#22c55e

    prdStatCd=01 은 판매상태가 아니라 신상품 표시 입니다.

    3-2. 자동 처리🔗

    Lonit은 새로 수집된 모든 상품에 prdStatCd=01 을 자동으로 붙입니다. 셀러가 신경 쓸 필요 없음.

    4. 상품명 — 100바이트 제한🔗

    11번가는 상품명을 UTF-8 100바이트까지 받습니다. 한글은 보통 3바이트 → 약 33자.

    4-1. Lonit 자동 절단🔗

    flowchart LR
    Source[원본
    '남자 빅사이즈 오버핏 코튼 후드 티셔츠 모음'
    123 bytes] --> Check{100 bytes 초과?}
    Check -->|예| Trim[단어 경계에서 절단
    '남자 빅사이즈 오버핏 코튼 후드 티셔츠'
    99 bytes]
    Check -->|아니오| Pass[그대로]
    Trim --> Apply[적용]
    Pass --> Apply

    style Trim fill:#fef3c7,stroke:#f59e0b

    자동 절단 시 단어가 잘려 의미 손실이 없도록 단어 경계에서 자릅니다.

    5. 브랜드 병기 — 노출에 영향🔗

    11번가는 브랜드를 상품명에 함께 표시하면 검색 매칭이 좋아집니다.

    "남자 빅사이즈 오버핏 코튼 후드 티셔츠"
                  ↓
"무신사스탠다드 남자 빅사이즈 오버핏 코튼 후드"

    Lonit은 자동으로 브랜드를 상품명 앞에 붙입니다. 단, 무신사·스탠다드 같은 금지어는 제외 (다른 마켓 정책 동일).

    6. 카테고리 — 4단계 + 검색 풀🔗

    11번가도 4단계 카테고리. Lonit이 자동 매핑.

    패션 > 남성의류 > 상의 > 후드티

    6-1. 카테고리 bulk 매핑🔗

    여러 상품의 카테고리를 한 번에 매핑할 때:

    상품 목록 → 다중 선택 → "카테고리 자동 매핑"

    이건 Lonit이 자동으로 한 번에 처리합니다 — 셀러는 다중 선택 후 "카테고리 자동 매핑" 버튼만 누르면 됩니다.

    7. 판매상태 — 등록 후 즉시 판매중🔗

    11번가는 등록 직후 판매중(selPrdStatCd=103) 상태로 들어가야 노출됩니다.

    flowchart LR
    Reg[등록] --> Default{기본 상태}
    Default -->|미설정| Wait[승인 대기 100
    노출 X]
    Default -->|sellPrdStatCd=103| Live[판매중
    노출 O]

    style Live fill:#dcfce7,stroke:#22c55e
    style Wait fill:#fee2e2

    Lonit은 등록 시 자동으로 sellPrdStatCd=103 을 보냅니다. 별도 확인 없이 바로 판매중으로 들어감.

    8. CS 처리 — 클레임 액션🔗

    11번가는 클레임(취소/반품/교환) 처리 API가 정교합니다. Lonit은 CS 통합 챕터에서 자세히 다룹니다.

    flowchart LR
    Customer[고객 클레임] --> Lonit
    Lonit --> Action{어떤 액션?}
    Action -->|승인| Approve[11번가 승인 API]
    Action -->|거부| Reject[11번가 거부 API]
    Action -->|회수 보류| Hold[회수 보류 API]

    style Lonit fill:#6366f1,color:#fff

    자세히는 6장.

    9. 판매기간 — 시작일/종료일🔗

    11번가는 판매 기간을 aplBgnDy (시작일) + aplEndDy (종료일) 형태로 받습니다.

    필드 Lonit 자동
    aplBgnDy 등록 당일
    aplEndDy 1년 후 (정책에서 변경 가능)

    10. 자주 발생하는 문제🔗

    10-1. "KC 인증 필요" 에러🔗

    해결: 정책에 KC 인증 정보 입력. 또는 KC 면제 카테고리로 재매핑.

    10-2. 상품명 100바이트 초과 거부 (해결됨)🔗

    현재는 자동 절단으로 해결됩니다. 그래도 발생하면 트러블슈팅 참고.

    10-3. "신상품 가중치 못 받는 듯"🔗

    prdStatCd=01 누락 가능성. 자동 적용되지만 수정된 상품의 경우 누락될 수 있음. 재등록으로 해결.

    11. 요약 체크리스트🔗

    11번가 노출 잘 되려면:

    • KC 인증 정보 정책에 입력
    • 카테고리 4단계 정확
    • prdStatCd=01 자동 (신상품 가중치)
    • sellPrdStatCd=103 자동 (판매중)
    • 상품명 100바이트 이내 (자동 절단)
    • 브랜드 병기 (금지어 제외)
    • 판매기간 자동 (1년 default)
