    주문 + CS 통합 관리🔗

    4개 마켓 주문을 한 화면에서. 송장도 한 번 등록하면 4마켓에 자동 분배.

    1. 주문 통합 — 셀러 시간을 가장 많이 줄이는 기능🔗

    flowchart TB
    subgraph 마켓들["🛒 4마켓에서 주문 발생"]
        N[스마트스토어 주문]
        C[쿠팡 주문]
        L[롯데온 주문]
        E[11번가 주문]
    end

    N -->|매분 동기화| Lonit[(Lonit 통합)]
    C -->|매분 동기화| Lonit
    L -->|매분 동기화| Lonit
    E -->|매분 동기화| Lonit

    Lonit --> View[📋 통합 주문 화면]

    View --> Tab1[📥 주문 받음]
    View --> Tab2[📦 발송 대기]
    View --> Tab3[🚚 배송 중]
    View --> Tab4[✅ 배송 완료]
    View --> Tab5[⚠️ 클레임]

    style Lonit fill:#6366f1,color:#fff,stroke:#4f46e5,stroke-width:3px

    더망고 시절: 4마켓 셀러센터를 각각 들어가 주문 확인. Lonit: 1개 화면에서 모든 마켓 주문.

    1-1. 주문 동기화 주기🔗

    마켓 동기화 주기
    스마트스토어 매분
    쿠팡 매분
    롯데온 매분
    11번가 매분

    새 주문이 들어오면 1분 안에 Lonit 화면에 나타납니다.

    2. 주문 화면 — 정보 한눈에🔗

    flowchart LR
    Order[주문 1건] --> Info{보이는 정보}
    Info --> A[주문번호 + 마켓]
    Info --> B[상품명 + 옵션]
    Info --> C[수량 + 가격]
    Info --> D[고객 정보
    이름·전화·주소]
    Info --> E[결제 상태]
    Info --> F[배송지]
    Info --> G[메모·요청사항]

    style Info fill:#fce7f3,stroke:#ec4899

    각 마켓마다 다른 형식의 주문 정보를 Lonit이 표준화해서 보여줍니다.

    2-1. 마켓별 표시 색상🔗

    flowchart LR
    A([🟢 스마트스토어]) --- 색1[초록]
    B([🔴 쿠팡]) --- 색2[빨강]
    C([🟣 롯데온]) --- 색3[자주]
    D([🔴 11번가]) --- 색4[붉은오렌지]

    화면에서 마켓을 색상으로 빠르게 구분.

    3. 송장 등록 — 한 번으로 4마켓🔗

    가장 강력한 기능. 

    flowchart TB
    Click[송장 등록 버튼] --> Input[택배사 + 운송장 번호]
    Input --> Detect{Lonit 자동 감지}
    Detect --> Market1[이 주문 = 스마트스토어]

    Market1 --> Send[해당 마켓 API에
    송장 자동 전송]
    Send --> Result[✅ 발송 완료 처리]

    Result --> Customer[고객 마이페이지에
    운송장 표시]

    style Detect fill:#fce7f3,stroke:#ec4899
    style Result fill:#dcfce7,stroke:#22c55e

    3-1. 일괄 송장 등록🔗

    여러 주문을 한 번에:

    1. 주문 목록 → 다중 선택
    2. 일괄 송장 입력 → 엑셀 붙여넣기
    3. 자동 분배

    엑셀 형식: 

    주문번호 | 택배사 | 운송장번호
2026-001 | CJ대한통운 | 1234567890
2026-002 | 한진택배 | 9876543210

    3-2. 다중 택배사 지원🔗

    택배사 코드
    CJ대한통운 CJGLS
    한진택배 HANJIN
    롯데택배 LOTTE
    우체국 EPOST
    로젠택배 LOGEN
    딜리박스 DELIBOX

    설정에서 자주 쓰는 택배사를 기본값으로 지정 가능.

    4. 클레임 처리 — 취소·반품·교환🔗

    stateDiagram-v2
    [*] --> 주문접수
    주문접수 --> 발송완료: 송장 등록

    주문접수 --> 취소요청: 고객 취소
    취소요청 --> 취소승인: 셀러 승인
    취소요청 --> 취소거부: 셀러 거부

    발송완료 --> 반품요청: 고객 반품
    반품요청 --> 회수보류: 회수 보류 검토
    반품요청 --> 반품승인: 회수 시작
    반품승인 --> 회수완료
    회수완료 --> 환불완료

    발송완료 --> 교환요청: 고객 교환
    교환요청 --> 교환승인
    교환승인 --> 재발송

    4-1. 클레임 액션 자동화🔗

    각 마켓의 클레임 처리 API를 Lonit이 자동으로 호출합니다.

    마켓 지원 액션
    스마트스토어 승인 / 거부 / 회수 보류 / 재배송
    쿠팡 승인 / 거부
    롯데온 승인 / 거부 / 보류
    11번가 승인 / 거부 / 회수 보류

    4-2. CS 답변🔗

    각 마켓의 고객 문의도 Lonit 한 화면에서.

    flowchart LR
    Customer[고객 문의] --> 마켓
    마켓 -->|매분 동기화| Lonit
    Lonit --> View[CS 화면]
    View --> Reply[답변 작성]
    Reply --> Send[해당 마켓에 자동 전송]

    style View fill:#fce7f3,stroke:#ec4899

    답변 템플릿을 미리 만들어두면 빠르게 응대 가능.

    5. 주문 처리 SLA🔗

    각 마켓이 셀러에게 요구하는 처리 시간:

    항목 스마트스토어 쿠팡 롯데온 11번가
    주문 확인 24h 이내 24h 24h 24h
    발송 결제 후 2영업일 2일 2일 2일
    CS 답변 24h 24h 24h 24h
    클레임 응답 48h 48h 48h 48h

    SLA를 어기면 셀러 점수 감점 → 노출 가중치 손실.

    Lonit의 알림 기능으로 SLA 임박 주문을 미리 알 수 있습니다.

    6. 일별 주문 현황🔗

    flowchart LR
    Daily[일별 주문 현황] --> Stats{통계}
    Stats --> S1[전체 건수]
    Stats --> S2[마켓별 건수]
    Stats --> S3[상품별 베스트]
    Stats --> S4[취소율]
    Stats --> S5[반품율]

    대시보드 → 매출 → 일별 현황 에서 확인.

    7. 자주 발생하는 함정🔗

    7-1. "송장 입력했는데 마켓에 안 나타남"🔗

    대부분 택배사 코드 mismatch. Lonit의 매핑 테이블을 확인하거나 트러블슈팅 참고.

    7-2. "주문이 Lonit에 안 들어옴"🔗

    마켓 API 키 만료 또는 IP 등록 문제. 설정 → 마켓 계정 → 연결 테스트.

    7-3. "클레임 답변이 마켓에 안 보임"🔗

    답변 후 마켓 동기화까지 1분 정도 걸림. 대기.

    8. CS 운영 베스트프랙티스🔗

    mindmap
  root((좋은 CS))
    빠른 응답
      24h 이내
      친절한 첫 인사
    명확한 정보
      배송일정 정확히
      반품 절차 안내
    공감
      불편 사과
      대안 제시
    기록
      모든 답변 보존
      반복 이슈 패턴화

    Lonit의 CS 화면은 답변 이력 자동 보존 + 반복 이슈 자동 패턴화를 지원합니다.
